这个项目计划做成一个用于项目状态检查的 Agent 主体。核心功能是读取并分析当前项目的各类材料，例如项目要求 PDF、代码、测试运行结果，以及其他需要提交的产品资料，然后对照要求检查整个项目的完成状态。首次运行可以先按完整检查流程设计，从头走完需求读取、代码核对、测试结果核对和报告输出。当前阶段可以先实现其中一条主流程：根据项目要求检查代码仓库生成的项目是否满足提交要求。

实现细节上，核心框架先基于 LangGraph 设计。Agent 会调用若干 tools，其中 PDF 识别是关键工具；本地文件读取、命令行执行、写文件等作为基础工具提供，后续可以接入已有的 PowerShell 实现。写入能力先收敛在报告生成上，也就是让 Agent 主要负责输出检查报告。其他工具还包括联网搜索，用于补充项目要求、依赖或外部资料的确认。

核心流程上，系统先拿到项目上下文，并结合本地可调整的系统提示词运行。由于使用 LangGraph，流程可以拆成多个步骤。以项目检查模式为例，第一步应明确项目要求，整合已有资料并生成对照式清单；之后按清单逐项检查代码、文档、测试结果和提交资料，最后汇总为报告。更细的节点边界、状态结构和工具调用策略后续再继续细化。
